# 科学 Coding 检查清单

## 修改前

- [ ] 是否阅读了相关文件？
- [ ] 是否理解了修改点附近的项目结构？
- [ ] 是否搜索过已有实现、工具函数、组件或接口封装？
- [ ] 是否确认接口、参数、返回值和行为真实存在？
- [ ] 是否明确任务目标？
- [ ] 是否明确修改范围？
- [ ] 是否明确验收标准或验证方式？
- [ ] 是否确认业务规则来自用户、文档或已有代码？
- [ ] 是否知道哪些文件、模块或行为不能改？
- [ ] 当前任务是可直接执行的小修改，还是需要先确认的大修改？

## 修改中

- [ ] 是否复用了项目已有模式？
- [ ] 是否保持了模块边界？
- [ ] 是否遵循命名、风格和架构规范？
- [ ] 是否只修改了任务需要的范围？
- [ ] 是否避免了不必要依赖？
- [ ] 是否避免了无意义抽象？
- [ ] 是否考虑了安全风险，例如密钥、权限、注入和数据泄露？
- [ ] 是否处理了空数据、异常输入、资源不存在、网络失败和权限异常？
- [ ] 是否避免了未经要求的历史行为变化？
- [ ] 是否只在必要时补充了注释或文档？

## 修改后

- [ ] 是否运行了相关测试？
- [ ] 是否运行了类型检查、Lint 或构建检查？
- [ ] 是否进行了必要的手动验证？
- [ ] 是否说明了完成内容？
- [ ] 是否列出了修改文件？
- [ ] 是否说明了验证结果？
- [ ] 是否说明了未验证内容？
- [ ] 是否说明了潜在风险？
- [ ] 是否避免了超出证据的完成声明？

## 必须停下来确认的情况

- [ ] 业务规则不明确。
- [ ] 需求有多种合理解释。
- [ ] 需要大规模重构。
- [ ] 会改变架构或模块边界。
- [ ] 会修改核心模块。
- [ ] 会改变数据结构。
- [ ] 验收标准不明确。
